# 政策與治理

本文件描述了 GitHub MCP 伺服器專案的政策與治理模型。

## 專案目標

GitHub MCP 伺服器的目標是提供一個強大、安全且可擴充的實作，用於模型上下文協定 (Model Context Protocol, MCP)，使其能與 GitHub 平台的各種功能進行互動。

## 治理模型

本專案由 GitHub 維護，並遵循 GitHub 的開源治理實踐。我們歡迎社群貢獻，並致力於維持一個開放、透明且協作的開發環境。

## 貢獻政策

所有貢獻都必須遵循專案的貢獻指南 (`CONTRIBUTING.md`) 和行為準則 (`CODE_OF_CONDUCT.md`)。

### 程式碼審查

所有程式碼更改都必須經過 GitHub 團隊成員的審查和核准，然後才能併入主分支。我們注重程式碼品質、測試覆蓋率以及對 MCP 規範的遵循。

### 測試要求

新的功能和錯誤修復必須伴隨著適當的單元測試和（如果適用）端對端測試。請參閱 `docs/testing.md` 了解更多關於我們測試實踐的資訊。

## 安全政策

我們非常重視專案的安全性。如果您發現任何安全漏洞，請按照 `SECURITY.md` 中詳述的流程進行報告。請不要在公開的 issue 中披露安全漏洞。

## 隱私政策

本專案遵循 GitHub 的隱私聲明。我們不會收集除了運行 MCP 伺服器所需的 GitHub API 呼叫之外的任何個人資訊。

## 授權

本專案根據 `LICENSE` 檔案中的條款授權。所有貢獻都必須在同一授權下發布。
